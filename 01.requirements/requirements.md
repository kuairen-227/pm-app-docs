# プロジェクト管理アプリ 要件定義書

## 1. プロジェクト概要

- 名称: プロジェクト管理アプリ
- 目的: 小規模チーム向けに、プロジェクト／タスク／ユーザー管理を最小限の機能で提供
- 想定ユーザー: 5〜10人規模のチーム
- 開発方針: 一人プロジェクト、MVPリリース後に拡張可能

## 2. 前提条件

- ユーザーはログイン必須
- 権限は管理者／メンバーで区別
- MVP段階ではガントチャートや通知機能は未実装

## 3. MVP機能範囲

| 優先度 | 機能 |
|--------|------|
| Must | ログイン認証、プロジェクト作成/編集/削除、タスク作成・状態更新 |
| Should | 権限管理（管理者/メンバー） |
| Could | 通知、リマインダー |
| Won't | ガントチャート、ラベル機能 |

## 4. 非機能要件

- 対応ブラウザ: Chrome, Edge 最新版
- データ保存: PostgreSQL
- 認証方式: JWT
- API仕様: REST / JSON
